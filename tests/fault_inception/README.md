# Тестирование определения момента начала КЗ (t0)

## Что тут лежит

```
tests/fault_inception/
├── README.md          ← этот файл
├── check_t0.py        ← скрипт ручной проверки
└── oscillograms/      ← сюда кладёшь свои .cfg + .dat файлы
```

## Как запускать

```bash
# Из корня репозитория:
python tests/fault_inception/check_t0.py
```

## Что делает скрипт

1. Сканирует `tests/fault_inception/oscillograms/` на наличие `.cfg` файлов.
2. Для каждого файла парсит COMTRADE, извлекает токи фаз A, B, C.
3. Запускает `detect_t0_multi_phase` из `data/fault_inception.py`.
4. Выводит в консоль таблицу:

```
Файл                     | t0 (отсчёт) | t0 (мс)   | fs (Гц)
-------------------------|-------------|-----------|--------
cutAB_5km_t10ms.cfg      |       500   |  10.000   | 50000
cutAC0_10km_t25ms.cfg    |      1250   |  25.000   | 50000
...
```

5. После таблицы — итоговый счётчик успехов/ошибок.

## Формат входных файлов

Обычный COMTRADE ASCII: `.cfg` + `.dat` в одной папке.  
Частота дискретизации берётся из `.cfg`-заголовка.

## Подстройка алгоритма

Если детектор промахивается, отредактируй параметры прямо вверху `check_t0.py`:

```python
PARAMS_OVERRIDE = dict(
    coarse_top_k     = 5,      # сколько пиков D4 рассматривать
    coarse_window_ms = 200.0,  # полуширина поиска вокруг грубой оценки, мс
    pre_fault_ms     = 20.0,   # не влияет на детектирование, только на обрезку
    post_fault_ms    = 60.0,
    threshold_mult   = 1.0,    # чем меньше — тем раньше срабатывает
)
```
